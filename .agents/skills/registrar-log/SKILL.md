# Registrar Log

## Objetivo

Registrar o que aconteceu em uma sessao para manter continuidade entre conversas e facilitar revisao futura.

## Destino

Adicionar uma nova entrada em `06 - Operations/Logs.md`.

## Workflow

1. Identificar objetivo da sessao.
2. Listar acoes executadas.
3. Listar arquivos criados, alterados ou movidos.
4. Registrar resultado.
5. Registrar pendencias.
6. Registrar aprendizados praticos.
7. Usar `[[wikilinks]]` para notas, projetos e conceitos relevantes.

## Formato

```markdown
## YYYY-MM-DD - Titulo da sessao

### Objetivo

### Acoes

### Arquivos Criados ou Alterados

### Resultado

### Pendencias

### Aprendizados
```

## Regras

- Ser conciso.
- Registrar fatos verificaveis.
- Separar pendencias de aprendizados.
- Evitar escrever log como resumo promocional.
- Incluir caminhos ou wikilinks quando isso ajudar a retomar o trabalho.
